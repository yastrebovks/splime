# YAML compatibility corpus

This corpus pins SPL YAML/IR documents produced by older public formats. The
files are intentionally static fixtures: tests must load these bytes from disk,
not regenerate them from current code, so serializer drift is visible.

`v02x/` contains 0.2.x/0.3.x-compatible documents generated on 2026-07-08 from
the 0.3.0 baseline with `ir_parse` / `get_top_level_deps` and
`yaml.dump_all`.

`v040/` is reserved for additive 0.4.0 documents such as adapter halves,
artifact tags, and node runtime tags. Add files there only when the
corresponding feature lands.

When adding a corpus case:

1. Generate the YAML once with the release version that introduces the format.
2. Save the generated bytes as a fixture with a header comment.
3. Add an expected execution result in the tests when the root is executable.
4. Do not generate corpus YAML inside tests.
